1. Method Overloading (Compile time polymorphism)

Same method name, different parameters, in the same class.

Example:

class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }

    double add(double a, double b) {
        return a + b;
    }
}

Usage:

Calculator c = new Calculator();

c.add(2, 3);
c.add(2, 3, 4);
c.add(2.5, 3.5);

Java decides which method to call based on the arguments.

Key point
Same method name
Different parameter list
Same class


2. Method Overriding (Runtime polymorphism)

Child class provides its own implementation of a method already defined in the parent class.

Example:

class Animal {

    void sound() {
        System.out.println("Animal makes sound");
    }
}
class Dog extends Animal {

    //@Override
    void sound() {
        System.out.println("Dog barks");
    }
}

Usage:

Dog d = new Dog();
d.sound();

Output:

Dog barks

The child's method replaces (overrides) the parent's version.

Key point
Same method name
Same parameters
Parent and Child classes


3. Another Example of Overriding

Parent class
class Calc {

    public int add(int n1, int n2) {
        return n1 + n2;
    }
}
Child class overriding the method
class AdvCalc extends Calc {

    //@Override
    public int add(int n1, int n2) {
        return n1 + n2 + 1;
    }
}
Usage
AdvCalc obj = new AdvCalc();

System.out.println(obj.add(2, 3));

Output:

6

because:

2 + 3 + 1 = 6
